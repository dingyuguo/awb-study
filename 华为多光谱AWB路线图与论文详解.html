0. 综述：从"猜色"到"测色"的双轨演进

华为在多光谱 AWB 上的布局，呈现出一条清晰的"双轨并行、最终交汇"的路线：一条是学术研究轨（华为芬兰研究中心、诺亚方舟实验室与高校合作发表的一系列论文），另一条是产业落地轨（从 RYYB 的偏色痛点出发，逐步演进到红枫多光谱影像系统）。两条轨道在 2024—2025 年交汇：学术上证明了"多光谱能解决 RGB 解决不了的颜色问题"，产业上则用红枫把这一结论做成了硬件。

一句话总览：华为的路径不是"突然在某一代手机塞进多光谱传感器"，而是先在论文里用严谨实验证明多光谱 AWB 的可行性（IJCV 2024），再提出 AWB+CCM 端到端联合优化的完整方案（WACV 2025），与此同时产品线上用色温/光谱传感器一步步从 5 通道走到 4×4 面阵（红枫），最终在 Mate 80 上以"Local AWB 全域色温感知"收口。

1. 技术背景：为什么需要多光谱 AWB

要理解华为的路线选择，先要厘清一个基本矛盾：传统 RGB AWB 在数学上存在不可克服的信息损失，而华为的 RYYB 路线又把这种损失放大了。

1.1 白平衡与颜色校正的分工（回顾）

相机 ISP 中，白平衡（WB）解决"光偏了"的问题——用对角矩阵把图像从未知光源变换到标准光源；颜色校正矩阵（CCM）解决"sensor 跟人眼不一样"的问题——用 3×3 全矩阵做 sensor→人眼的色彩映射。两者必须先 WB 再 CCM，先后不可颠倒（详见配套文档《白平衡与光源概念辨析》）。

1.2 RGB 的根本局限：同色异谱（Metamerism）

图像形成模型（第 k 通道）：

f_k(x) = m(x) · ∫ e(λ) · s(x,λ) · ρ_k(λ) dλ , k ∈ {R, G, B}

其中 e(λ) 是光源 SPD，s(x,λ) 是物体反射率，ρ_k(λ) 是 sensor 第 k 通道的光谱灵敏度。RGB 只有 3 个通道，等于把连续光谱投影到 3 维空间。不同形状的真实光源，只要在这 3 个投影上相同，就会得到完全相同的白平衡点——这就是同色异谱。

这带来两个致命问题：

  • 无灰点 / 干扰色（dominant-color）场景失效：当场景被单一颜色主导（如大面积红布、纯色舞台灯），传统基于"灰世界/白patch"假设的统计法失去参照，白平衡崩溃。
  • 无法区分 LED 与日光等光谱差异大的光源：它们在 RGB 响应上可能投影相同，但真实色彩还原效果迥异。

1.3 多光谱如何破局

把通道数从 3 提升到 C（如 10–16），就把"光源→sensor 空间"的映射维度从 3 维拉到 C 维。在 C 维空间里，同色异谱光源的投影大概率可分，于是：

  • 白平衡点不再只是 3 维摘要，而是 C 维光源响应（在 sensor 空间内，白点 = 光源的逐元素倒数，两者信息等价）；
  • 少数"关键波段"即可构成光源的"光谱指纹"，无需完整光谱；
  • 从 C 维恢复连续 SPD 是一个近似良态（well-posed）的逆问题，而 3 维→SPD 不可逆。

1.4 关键区分：单点（single-pixel）vs 面阵（area / snapshot）多光谱

类型代表是否参与成像输出在 AWB 中的角色
单点多光谱传感器IJCV 2024 的虚拟光谱探头；早期手机色温传感器否（仅测光）1×C 场景平均光谱估全局光源 / 白点
面阵多光谱（snapshot SFA）红枫 1.0/2.0；三星 Beyond RGB；OPPO 丹霞可成像（但手机里多作"色彩总监"辅助测光）H×W×C 光谱图像估全局 + 局部（Local AWB）光源
重要概念澄清：手机里的面阵多光谱（如红枫）与科研用"真·多光谱成像"（用于物质分析、医学）目标不同——前者是辅助测光、不参与主成像，后者追求成像本身。这是工业界与学术界"多光谱"的核心区别，也决定了华为路线是"用光谱测光来校准 RGB 主摄"，而非"用光谱直接出图"。

2. 学术路线：论文详细解读

华为在多光谱 AWB 上的学术贡献，目前可确认有两篇核心论文（均发表于华为研究中心/诺亚方舟实验室与高校的合作框架下），以及若干支撑性专利。下面按阶段逐篇拆解。

论文① Single Pixel Spectral Color Constancy（单像素光谱颜色恒常性）

BMVC 2021（会议） IJCV 2024（期刊扩展） Huawei Finland + 坦佩雷大学 第一作者 Samu Koskinen 通讯/二作 Erman Acar（华为）

原文：Int. J. Comput. Vis. 132:287–299 (2024), doi:10.1007/s11263-023-01867-x；会议版见 BMVC 2021。

2.1.1 背景与动机（三步逻辑）

论文开篇点明：颜色恒常性仍是相机色彩处理的最大挑战之一，CNN 虽有改善，但在单一颜色主导（single-color-dominant）场景下仍大量失效。作者换了一个设定发问："如果我们有 RGB 之外的别的信息，且能在手机上实现，什么信息最能帮上忙？"其动机分三步：

  1. RGB 的局限：仅 3 通道不足以表征光源，无法区分 LED 与日光等光谱差异大的光源，干扰色场景尤其易失效。
  2. 场景平均光谱 ≈ 光源光谱：用单点多光谱传感器测到的"物体反射率 × 光源光谱 × sensor 灵敏度"的积分，其形态与真实光源光谱高度接近（论文图 1：黑实线=真实光源，虚线=传感器测得平均光谱）。即"通过场景平均光谱推测光源是可行的"。
  3. 关键通道 = 光谱指纹：用"逐通道剔除看误差变化"的实验发现，某些特定通道一旦缺失，误差飙升 30%–300%——少数关键波段即可承担区分光源的主要作用。

2.1.2 核心发现

  • 通道数：仅需 10–14 个光谱通道即可取得最优效果（少于 10 退化，多于 14 收益递减）。
  • 数据量优势：光谱传感器输出比 10MP 手机的 RAW-RGB 少 5 个数量级，却捕获了光源的"光谱指纹"。
  • 可扩展多光源：虽完全无空间信息，但在多光源混合下也展现 promising 结果。

2.1.3 方法：如何"无中生有"造出多光谱训练数据

由于手头只有 RGB 数据集，作者设计了一套 RGB-to-Spectral 转换流程（基于 Sensor-to-Sensor Transfer 模型），把现有 RGB 颜色恒常性数据集"升维"成光谱数据。共四步：

① 光源光谱估计 L → L̂_spec （从 100 条候选光谱库中找最佳匹配，并做以 530nm 为支点的红/蓝线性微调，完美贴合 Cube+ 的 GT 白点）
② Raw→光谱图像 I_raw → R̂_spec （用 Munsell 反射率库反解物体反射谱，31 维 400–700nm / 10nm）
③ 光谱图像精修 R̂_spec → R̂′_spec
④ 平均反射光源的 sensor 采样 (R̄′_spec · L̂′_spec) → s （得到 1×C 的"单像素光谱测量向量 s"）

之后是白点回归（White Point Regression）：把 s 作为输入，用回归函数 l = f(s) 直接预测 RGB 白点 l=(l_R,l_G,l_B)。作者对比了三种回归器：Kernel Ridge（KR）、Random Forest（RF）、MLP，结论是三者差异很小，任选其一皆可。MLP 为 3 层全连接（512-1024-512，Adam）。

2.1.4 传感器设计与噪声模型

  • 测试了 21+2 种传感器配置，探索通道数、带宽（FWHM）、中心波长如何取舍。
  • 通道带宽与信噪比权衡：宽通道 SNR 高；作者设定极低光条件（最灵敏通道光子数 = 20×带宽 nm），用泊松噪声模型 s = s + √(20W·s) 验证鲁棒性。

2.1.5 数据

  • 生成数据：基于 Cube+ 数据集（含 GT 白点），按上述流程生成大规模光谱训练样本；多光源变体共 83,000 个样本。
  • 真实数据：自采 235 张 RAW 图（华为 Mate 20 Pro 拍摄）+ Konica Minolta CL-70F 光谱仪测得的场景平均光谱与 GT 光源。刻意包含"少数主色、无清晰灰区"的困难场景。

2.1.6 实验与结果

  • 单数据集（in-dataset）：显著优于传统 SOTA（约 2 倍提升），已接近数据集偏置上限。
  • 跨数据集（cross-dataset，仅在 Cube+ 训练）：在未见测试集上同样大幅领先，困难场景平均提升 40%–70%。
  • 多光源：结果 promising，证明即便无空间信息，光谱传感器也能同时检测多个光源的"光谱指纹"。

2.1.7 评价（业界解读的 Pros / Cons）

优点

  • 首次从实验角度严谨证明：光谱 AWB 在 corner case（无灰点）下确实有效；
  • 写作环环相扣，motivation 与技术紧密咬合；
  • 效果"炸裂"，且接近数据集偏置上限，说明方法已逼近该类数据的极限。

局限 / 争议

  • 运算量：对真实 RAW 需先反推光谱，开销大；
  • 泛化：在真实光谱数据集上效果明显下降，且 14ch 与 65ch 之间差距过大，说明真实数据对通道数很敏感；
  • 未开源：工业合作论文不开源，复现与验证受限。
对本路线的意义：这篇论文是华为多光谱 AWB 的理论奠基——它把"为什么多光谱能解决 RGB 解决不了的颜色问题"讲清楚了，并给出"10–14 通道即可"这一关键工程指引。它直接为后来红枫"8–9 通道 / 12 通道"的硬件选型提供了学术背书。

论文② Multi-Spectral Image Color Reproduction（多光谱图像颜色复制）

WACV 2025 华为诺亚方舟实验室 + 中科大 一作 Jiacheng Li（中科大，诺亚实习） 通讯 Zhiwei Xiong（中科大）/ 华为 Chang Chen 等

原文：Li et al., WACV 2025, pp.4344–4354（CVF Open Access）；IEEE DOI: 10.1109/WACV61041.2025.00814。

2.2.1 背景与动机

论文目标是从 RGB 颜色复制升级到 MSI（多光谱图像）颜色复制。作者指出一个反直觉的发现：单纯提高光谱分辨率（更多通道）并不自动带来颜色恒常性——直接把 MSI 投影到标准空间反而会导致"欠饱和（undersaturation）"。因此如何"用好"多出的光谱信息才是关键。

核心动机：传统 ISP 把 AWB 与 CCM 分两步独立做，误差会逐级传播；而多光谱数据既然包含了更完整的光谱信息，应该让 AWB 和 CCM 联合优化，而非"先估光源再查表"。

2.2.2 方法：正则化色彩管线（Regularized Color Pipeline）

标准 RGB 颜色复制是 I_RGB = I_MSI · T（T ∈ ℝ^{C×3}）。论文把 T 的求解拆成三个受正则约束的阶段：

Stage 1 — RGB 子空间投影（降维）：
  T 在"传感器光谱灵敏度(SSF)引导"下做端到端优化，找到一个 sensor-optimized 的 RGB 投影，兼顾"颜色匹配"与"后续恒常性"；
Stage 2 — 白平衡（WB）：用色度适应神经网络估计场景照度 L（维度 C×1，Angular Error loss）；
  3 维光源由 Illuminance Projection 得到： l_3 = T · L ；
Stage 3 — 颜色校正（CCM）：网络估计一个照度自适应（illuminance-adaptive）的 CCM。

关键设计是粗到细（coarse-to-fine）+ 残差 CCM：先用一个独立网络在虚拟数据集上预标定一个 coarse-CCM，主网络只回归 CCM 的残差（res-CCM），用 RMSE loss 监督。这样做"训练更易收敛、使用端侧可动态生成 CCM"。

2.2.3 多光谱数据增强（解决真数据稀缺）

真实多光谱数据获取昂贵耗时，作者提出色彩→反射率检索字典（color-to-reflectance retrieval dictionary）策略：从海量现有高质量 sRGB 图像中检索/生成高光谱反射率数据，从而能在大规模合成数据上预训练，提升泛化与鲁棒性。

2.2.4 实验与结果

  • 在三种典型光源下，多数情况角误差 < 1°，平均色差 ~2 ΔE，接近人眼辨色阈值；
  • 在合成与真实数据集上均超越对比方案；
  • Fig.2 直观展示：更高光谱分辨率（MSI）在把 RAW 投影到 CIE XYZ 时，色卡点在 CIE 1976 UCS 色度图上明显更贴近参考点。
与本路线的意义：这篇论文把"多光谱 AWB"从阶段①的"估白点"推进到"直接出标准色彩空间（CIE XYZ）+ AWB/CCM 联合"。它是红枫 2.0"Local AWB 全域色温感知 + 动态色彩校正"的直接学术原型——既然 CCM 可以随照度动态生成，那么在不同区域用不同 CCM 做局部校正（Local AWB）就顺理成章。

3. 产业路线：产品落地时间线

华为的多光谱之路，起点其实是一个痛点——RYYB 传感器为了进光量牺牲了颜色准确性。下面按代际梳理硬件与算法的演进。

3.1 起点：RYYB 的"原罪"与颜色痛点

2018 年华为与索尼联合研发 IMX600，在 P20 Pro 上豪赌 RYYB 阵列（2 黄 1 红 1 蓝，黄像素同时透过红+绿），进光量较 RGGB 提升约 40%，奠定夜拍优势。但代价是：最终合成彩色仍需红蓝绿三色，绿色要靠算法从 Y 里"猜"出来——对 ISP 算力要求极高，且一翻车就偏色。

历史佐证：P20 Pro 用户与 DxOMark 均反映"白平衡不准、室内偏黄/粉、拍白偏红"。这并非个例，而是 RYYB 在"颜色准确性"上的结构性短板——华为自己也承认"想优化好 RYYB 对算法端的挑战确实挺大"。这条痛点，正是后来多光谱路线的根本动因。

3.3 红枫到底做了什么：从"猜色"到"测色"

传统手机主摄（RGGB 或 RYYB）只采红绿蓝三波段，其余光谱只能靠白平衡算法"猜"。红枫是一颗微型光谱仪：

  • 它捕捉环境的二维光谱分布图像（面阵多光谱），而非单点；
  • 将光谱信息"融合"进主摄/超广角/长焦的真实成像，对不同分区做精准色彩调节；
  • 对纯色、高饱和、混合光源等 RGB 容易翻车的场景，直接"测"出光谱成分，避免单波长光干扰色温判断——这正是 IJCV 2024 所论证的"光谱指纹"思想。

5. 两条路线的交汇：学术如何驱动红枫

产品代际对应的学术思想映射关系
红枫 1.0（Mate 70，2024） IJCV 2024：单点/平均光谱 → 估光源白点；光谱指纹；10–14 通道足够 用面阵多光谱"测量"环境光谱（而非算法猜），对应论文"多光谱可解决无灰点/干扰色 AWB"；8–9 通道的硬件选型呼应"10–14 通道即可"的结论。
红枫 2.0（Mate 80，2025） WACV 2025：AWB+CCM 联合；照度自适应 CCM（粗到细/残差）；全域感知 从"全局白点"升级到"Local AWB 全域色温感知"——不同分区用不同校正，正是 WACV 2025 "动态生成 CCM、随照度自适应"的落地；30–40 TOPS 算力使高维光谱解析与多风格并行可在端侧实时完成。

一句话：红枫 1.0 解决了"能不能测得准光源"，红枫 2.0 解决了"测准之后怎么分区用好"——恰好对应论文①（估光源）与论文②（联合优化 + 自适应校正）的两步跨越。

6. 领域全景与未来方向（重点：Local AWB）

6.1 横向对比：同期的关键工作

工作机构贡献与华为的关系
Beyond RGB（WACV 2024） 三星 首个移动端真实多光谱数据集：16 通道、648×484、1680 场景，附分光光度计测得的 36 维真实光谱 GT；提出 ISE 算法超 SOTA 58.6%。 领域基准。华为路线依赖类似真数据，但华为更强调"辅助测光"而非"真成像"，两者场景假设不同。
Leveraging Multispectral Sensors for Color Correction（CVPR 2026） Bianco 团队（米兰比可卡大学）+ Vazquez-Corral 高分辨率 RGB + 低分辨率 MS 双传感器端到端联合色彩校正，跳过显式 AWB 损失，直接出色彩准确结果，误差较 baseline 降最多 50%。 与 WACV 2025 思想同源（AWB+CCM 联合），但非华为工作，属学术竞品。
CIC33（本地论文，用户已下载） — 多光谱用于 Local AWB / 多光源：MSI 相比 RGB 角度误差降 29%、ΔE2000 降 25%；验证大规模模拟数据泛化性优于中等真实数据。 补足华为路线中"局部/多光源"这一块——正是红枫 2.0 Local AWB 的学术参照。
Revisiting Image Fusion for Multi-Illuminant WB（ICCV 2025） Vazquez-Corral 等 多光源白平衡的图像融合方法。 Local AWB / 多光源方向的同期热点。

6.2 Local AWB：华为路线的下一站

Local AWB（局部自动白平衡）指不再用"一张图一个全局白点"，而是对图像不同区域估计并应用不同光源/校正。这是多光谱 AWB 的天然延伸，也是华为红枫 2.0 已点题的方向：

  • 为什么多光谱特别适合 Local AWB：面阵 MSI 本身带有空间维（H×W×C），可以在每个空间位置做高维光源估计，避开了 RGB 因同色异谱而无法做局部区分的困境；
  • 当前进展：红枫 2.0 的"全域色温感知"已具备局部能力；学术上 CIC33 证明了 MSI 在 Local AWB 上相对 RGB 的显著增益（角度误差 −29%、ΔE2000 −25%）；
  • 待解难题（来自论文反思）：
    • 信噪比：窄带多光谱通道 SNR 低，暗光/低反射场景可能失效、影响稳定性；
    • 通道数敏感：IJCV 2024 显示真实数据对通道数极敏感（14ch vs 65ch 差距大），硬件选型需谨慎；
    • 真实数据泛化：论文①在真实光谱数据上效果明显下降，说明模拟数据训练的泛化仍是瓶颈（CIC33 同样指出：大规模模拟 > 中等真实）；
    • 多光源分解：如何把混合光源在局部准确拆分（Slot Attention / 图像融合等）仍是开放问题。

6.3 趋势判断（综合业界解读）

业界对 AWB 未来五年的判断集中在三条路径，华为已占其中两条：

① 多光谱 / 面阵（华为已量产）

基于第一性原理的"测光"，能根本解决纯色、多光源、模组不一致等核心问题；量产上去后迭代快。

② 生成式模型革新 ISP

直接对 coarse sRGB 做增强（如长焦人像肤色用大模型而非前端 AWB）。华为 XMAGE + 鸿蒙 AI 已在此布局。

③ 多模态 AWB

融合色温传感器、曝光、ISO、时间/位置、姿态甚至深度等。华为多传感器融合（红枫 + 主摄 + 环境参数）天然契合。

用多光谱增强局部自动白平衡

Enhancing Local Automatic White Balance with Multi-Spectral Imaging — CIC33 (2025)

这是一篇来自 Spectricity（比利时多光谱摄像头公司） 的会议论文。它回答了一个很朴素但很难的问题：在一个场景里，如果不同地方被打上了不同颜色的光，相机怎么给每个地方分别"校白"？ 论文的答案是：给相机多加几个"窄带光谱通道"（多光谱），让网络在更丰富的光谱信息上做局部白平衡，并用大规模"模拟数据"训练，从而比传统 RGB 方法更准、更稳、更省算力。

如果你对"白平衡/多光谱/局部白点"完全陌生，这章把全部前置知识补齐。已经懂的读者可以直接跳到第 1 章。

0.1 光、颜色、人眼与相机

我们看到的"颜色"，本质上是物体反射的光进入眼睛后的感受。太阳光或灯泡发出的光，包含从约 400nm（紫）到 700nm（红） 的连续波长（可见光光谱）。一个红苹果之所以"红"，是因为它吸收了大部分波长、只把 ~600–700nm 的光反射出来。

人眼有三种视锥细胞（L/M/S），分别对长/中/短波长敏感，大脑把这三种信号合成为"颜色"。相机则用一个 拜耳滤镜（Bayer CFA） 把每个像素分成 R/G/B 三个宽通道来近似人眼——这就是普通 RGB 相机的来历。注意：相机的 R/G/B 三个通道各自覆盖一大段波长（"宽带"），并不是单一波长。

0.2 什么是白平衡（颜色恒常性）

同一个白纸，在黄昏（偏黄）的窗边和在荧光灯（偏蓝绿）的办公室里，相机拍出来的 RGB 值完全不同。但人眼会"自动脑补"：不管什么光，白纸看起来还是白的。这种能力叫 颜色恒常性（color constancy），相机里对应的模块叫 自动白平衡（AWB）。

AWB 的标准做法分两步（论文引言原话）：

AWB 的两步定义（论文原文）

① 检测光源的"白点"（white point）：即当前光是什么颜色；
② 把该光源替换成标准光源（如 D65，正午日光）：用对角矩阵（每个通道乘一个增益）把图像"校白"。

白点本质上就是"光源颜色"在相机通道上的投影（一个 3 维向量，对 RGB 相机）。"校白"就是算出"当前光源 → 标准白光"的比值（对角矩阵），乘到图像上约掉光源。我们在之前的《白平衡与光源概念辨析》里详细推导过这层关系——这里只需记住：白平衡 = 估光源 → 校正到标准光源。

0.3 成像模型：一张照片是怎么"算"出来的（重点公式）

理解这篇论文，最关键的是下面这个成像模型。对场景中一个"点"（或一小块），相机记录到的测量值 m 由三部分相乘得到：

m = S · Λr · λ
// m : 相机测到的光谱测量值（RGB 是 3 维，MSI 是 15 维）
// S : 相机的光谱灵敏度矩阵（每一列是一个通道对不同波长的响应）
// Λr : 物体反射率的对角矩阵（对角线上是反射光谱 r(λ)）
// λ : 光源的光谱功率分布 SPD（不同波长的光强）

直觉：光 λ 照到物体上，物体只反射 Λr 那部分，再被相机用 S "看一眼"，得到测量 m。问题来了——给定 m 和 S，我们想还原物体的真实颜色 r，但 λ（光源）也是未知的。两个未知数、一个方程，这是一个欠定（ill-posed）问题。这就是白平衡之所以难的根本原因。

同色异谱（metamerism）：RGB 的死穴

两个不同的光源（比如某 LED 和某荧光灯），只要它们经过相机 3 个 RGB 通道的积分结果一样，就会让相机拍出完全相同的 RGB 值。但物体在它们下面真实的颜色还原可能完全不同。RGB 只有 3 个宽通道，信息量太少，分不清这种"同色异谱"的光源。这正是多光谱的切入点。

0.4 全局 WB vs 局部 WB（Local AWB）vs 多光源

🌍 全局 WB（传统）

整张图假设只有一个光源，估一个白点，全图乘同一个对角矩阵。简单，但遇到混合光就翻车。

📍 局部 WB / Local AWB

场景里不同位置可能有不同光源（窗边日光 + 室内灯光）。局部 WB 要给每个区域/每个像素估一个各自的白点，输出一张"白点图"（illumination map），而不是单个白点。

局部 WB 比全局 WB 难得多：每个局部区域颜色可能很少（比如一大块红墙），用来估光源的线索极少，歧义更大。论文的核心卖点就是：多光谱的窄带信息能大幅降低这种局部歧义。

0.5 为什么 RGB 不够，多光谱为何破局

多光谱相机（如本文的 Spectricity S1-M）不是 3 个宽通道，而是 15 个窄带通道（每个通道只"看"一小段波长）。这带来两个好处：

  • 光谱指纹：不同光源在 15 个窄带上的"形状"差异明显，几乎可以唯一区分（华为 IJCV 2024 首次系统证实：场景平均光谱形态 ≈ 光源光谱）。
  • 近似可逆：3 维 RGB → 真实光源 SPD 不可逆（无穷多 SPD 投影到同一点）；而 15 维 MSI → SPD 近似可逆（见第 2.4 节的 LSMI 成像模型分析）。

0.6 怎么衡量"校白校得好不好"：两个指标

评价指标（论文全程使用）

① 白点角度误差（white point angular error，单位：度）：把预测白点和真值白点都当成单位向量，算它们之间的夹角。
小角度 = 估得准。论文报 mean（平均）和 median（中位数）。

② 颜色准确度 ΔE2000：白平衡之后，用 Macbeth 色卡算预测颜色与标准色的差异。ΔE 越小越准（一般 <3 人眼难以察觉，<6 尚可）。

一句话贡献

提出一个基于多光谱（MSI）的局部白平衡方法：用一个参数量很小（约 2M）的 Set Transformer 网络，在大规模模拟多光谱数据上训练，在局部白点估计上 与 SOTA 持平（LSMI 上）且泛化更好（自有验证集上超越 SOTA），并证明 多光谱比 RGB 更准（角度误差↓约 29%，ΔE↓约 25%）。

三个核心贡献点

  • 方法：一个简单、轻量、permutation-invariant 的 Set Transformer 网络，可做 RGB 版也可做 MSI 版。
  • 数据：用"模拟测量"训练（绕开真实数据稀缺），并提出 MIMI 验证集（774 场景，RGB+MSI 双摄像头配对，白卡真值）。
  • 发现：① 模拟数据训练的泛化性远好于中等规模真实集（LSMI）训练的；② MSI 比 RGB 局部白点更准。
⚠️ 论文在领域里的位置

2 方法逐节精讲（论文 "Methods and data" 章）
这一章是论文的硬核。我们按"训练数据 → 网络 → 验证数据 → LSMI 模型批判"四节展开，并放大两个最该讲清的点：网络（Set Transformer）与 LSMI 成像模型的近似误差。

2.1 训练数据集：用"模拟测量"绕开真实数据稀缺

论文开篇就点明：要训好局部白平衡，需要大量带"局部白点真值"的多光谱图；但这种数据集目前不存在（真实测量无法获得逐像素密集白点真值）。所以作者选择模拟生成训练数据。

模拟数据怎么造？

在"完美朗伯体（Lambertian，即理想漫反射，无高光、无互反射）"假设下，每个测量 = 一个 光源 SPD 与一个 反射光谱 相乘后，投影到相机敏感度 S 上。作者准备两个大数据库：

💡 光源数据库

32768 个 illuminant SPD，涵盖荧光灯、白炽灯、卤素灯、LED，以及不同大气条件下的日光，还有它们的混合。

🎨 反射率数据库

6814 个 reflectance 光谱（公开 + 内部测量），代表各种物体表面颜色。

一个训练样本（set）的生成协议

论文说"典型大小 128"——即每个训练样本是 128 个测量的集合。生成步骤如下（这是复现的关键，见第 4 章）：

// 伪代码：生成一个 set（128 个 measurement）
1. 从 32768 个光源中选 1 个 或 多个光源；
2. 若多个光源 → 随机生成混合系数，拼成该 measurement 的"专属光源"；
3. 随机采样 1~15 个反射率光谱，分配给该 measurement 的像素；
4. measurement = (光源 SPD) × (反射率) 投影到相机 SSF（S1-M 得 15 维，Nikon 得 3 维）；
5. 真值白点 = 该 measurement 的光源 SPD 投影到同一 SSF 并归一化；
6. 每个光源（或混合）共造 16 个 set → 总训练集 = 524288 个 set。

关键点：约一半 set 用单光源，一半用混合光源（对应多光源/局部场景）。而且每个 epoch 都重新打乱光源顺序、重新采样反射率——也就是说，每个 epoch 喂给网络的模拟样本都不一样，这是一种"无限数据增强"，有效防止过拟合。

🤔 模拟数据的局限（论文自己承认）

这个模型很简化：① 不考虑真实图像里光源/反射率的空间与光谱相关性；② 不考虑朗伯体之外的复杂光-物质交互（高光、互反射、次表面散射）。但论文用实验结果证明：即便这么简化，在真实数据上仍能取得有竞争力的局部白点估计。这正是"大规模模拟 + 轻量网络"路线的底气。

2.2 网络架构（重点！为什么是 Set Transformer）

论文说：现有 local WB 方法用的都是大模型（如 LSMI-U 有 56M 参数），超出手机算力。所以他们用了一个相对简单、基于 Transformer、参数很少 的网络。整个网络由 4 个 block 串起来：

图：CIC33 网络四块结构。输入是无序集合（128 个测量），输出是同序集合（128 个白点）。
  • ① Encoder：简单的全连接前馈网络（feed-forward），独立地处理每个测量（不跨测量交互），把它编码成一个 embedding。
  • ② Aggregation（聚合）：用 Set Transformer 从 128 个 embedding 里提炼出全局的"光源信息"。
  • ③ Dispersion（分发）：再用 Set Transformer 把提炼出的全局光源信息回灌到每一个 embedding 上（让每个点都知道"全局大概什么光"，再结合自身线索估自己的白点）。
  • ④ Decoder：简单全连接，把每个 embedding 解码成该点的白点（15 维，归一化）。

为什么非得是 Set Transformer？关键性质：置换不变（permutation-invariant）

输入是一"集合"（set）的 128 个测量，没有顺序、没有空间结构。如果换个打乱顺序喂进去，输出（对应每个点的白点）也应当只是 correspondingly 打乱，而不应改变。普通 CNN/RNN 对顺序敏感，不适合；Set Transformer 天然置换不变，完美契合。而且它用"诱导点（inducing points）"把注意力复杂度从 O(n²) 降到 O(n·m)，所以能塞进手机。

Set Transformer 核心公式（论文 backbone [12]，详见第 5.1 节）

MAB(A,B) = LayerNorm( A + rFF( LayerNorm( A + MHA(A,B) ) ) )  // 多头注意力块
SAB(X) = MAB(X, X)  // 自注意力
ISAB(X) = MAB( X, MAB( I, X ) )  // I 是可学习的"诱导点"，实现置换不变 + 降复杂度
PMA(X) = MAB( I, SAB(X) )  // 池化聚合

两种变体 + 微调

  • RGB → RGB：输入 RGB 测量，预测该 RGB 相机的白点。
  • MSI → MSI：输入 15 通道多光谱测量，预测多光谱白点（本文主推）。
  • MSI → MSI → RGB：先估 MSI 白点，再转换到某 RGB 相机（用于跨相机对比）。
  • 也可在真实数据上 fine-tune。

训练设置：100 epochs，初始学习率 1e-4，若验证损失 5 个 epoch 不降则减半；数据增强包括光源 SPD 的小随机形变 + 给模拟测量加真实 shot noise（散粒噪声）。

2.3 验证数据集 MIMI：怎么拿到"真值"

模拟数据只能训练，验证必须靠真实数据。但论文认为 LSMI 数据集的"密集白点图"不够准（见 2.4），所以自己采了一个 MIMI（Multi-Illuminant and Multi-spectral Image）验证集：

  • 774 个场景，每个场景一对图：一张无白卡（用于跑算法），一张有白卡（用于量真值）。
  • 硬件：把 Samsung Galaxy Tab S9（RGB） 与 Spectricity S1-M VIS（15 通道 MSI 快照模组） 并排固定在三脚架上（保证两图严格对齐）。
  • 白点真值：在场景中平面上摆白卡，第二张图量白卡处的多光谱值即该处真值白点；用 Macbeth 色卡 量白平衡后的 ΔE2000。
  • 关键优势：因为 MSI 可精确变换成任意 RGB 相机的敏感度（见图 6），所以 MIMI 能给"任何 RGB 相机"生成白点真值——这是它比 LSMI 更灵活的地方。

2.4 LSMI 成像模型的近似误差（论文最锋利的分析）

论文花了整整一节，指出 LSMI 数据集用来生成"密集白点图"的成像模型，对 RGB 相机是不准确的。这是理解"为什么 LSMI 训练会过拟合/泛化差"的理论根因。推导如下：

真实模型（表达式1）:   m = S · Λr · λ
LSMI 近似（表达式2）:
  m ≈ S · Λr · S+ · S · λ
    = Dr · λ   // Dr = S·Λr·S+
    = dr ⊙ λ   // dr = Dr·1（把 Dr 当成对角阵，取对角线）

这个近似成立，必须同时满足两个假设：

假设 A

光源 SPD λ 完全落在 S 的列空间里（即 S⁺Sλ = λ）。对 RGB（S 只有 3 列）几乎不成立——真实光源是高维曲线，3 个基张成的子空间装不下。

假设 B

Dr 是对角矩阵。但 Dr = S·Λr·S⁺ 一般不是对角的（通道间有串扰）。对 MSI（15 通道，S 接近满秩）则近似成立。

💡 一句话结论

LSMI 的密集白点图建立在"RGB 也能精确反解光源"的近似上，而这个近似对 RGB 相机很差、对多光谱相机才好。所以 LSMI 标出来的"真值"本身就有系统误差，用它训练/验证自然泛化差——这也解释了第 3 章 Table 3 里 LSMI-U 在 MIMI 上崩掉的现象。

核心洞见

训练数据规模与多样性 > 网络大小。中等规模真实集（LSMI）会过拟合；大规模、覆盖广的模拟数据（524288 set，含混合光源、噪声增强）泛化性最好。这呼应了我们《Local AWB 待解难题》里"数据瓶颈"那一节。

解读：

  • MSI → MSI 比 RGB → RGB：角度误差 2.8→2.0（降约 29%），ΔE 4.8→3.6（降约 25%）。这是论文"多光谱助力局部白平衡"的直接证据。
  • 把 MSI 白点转到 RGB 域（MSI→MSI→RGB）略差（2.3），源于交叉相机差异和转换不完美（尤其荧光灯）。
  • MSI 方法 ΔE 特别低，因为"用白点从 MSI 测量里移除光照"比在 RGB 里更精确（见 0.5 节的"近似可逆"）。

3.4 综合结论

① 小网络在 LSMI 上能打平大模型；② 模拟训练泛化远胜 LSMI 训练；③ 多光谱比 RGB 局部白点更准、颜色更真。三点共同支撑论文标题：Enhancing Local AWB with MSI。

4.1 复现总览（需要什么）

📦 数据集

① 模拟训练集（自生成）；② MIMI 验证集（需硬件采集，或用其公开版）；③ 相机 SSF（S1-M 15通道 / Nikon D810 3通道）。

🧠 网络

Set Transformer 四块，~2M 参数。PyTorch 即可。

🎯 Loss

论文未明说；推断为 angular loss（见 4.4）。

🏃 训练

100 epoch，lr 1e-4 减半，SPD 形变 + shot noise 增强。

4.2 数据集准备

4.2.1 模拟训练数据生成（核心步骤）

# 伪代码（PyTorch 风格，单个 set 的向量化生成）
SPD_DB = load('illuminants_32768.npy') # (32768, L) L=波长采样点数
REF_DB = load('reflectances_6814.npy') # (6814, L)
S = load('S1M_sensitivity_15ch.npy')# (15, L) 相机敏感度

for each set (128 measurements):
  n_illum = 1 if rand()<0.5 else 2..K # 半单光源半混合
  illum = mix(SPD_DB.sample(n_illum), weights) # 混合成专属光源 (L,)
  for each of 128 measurements:
    r = REF_DB.sample(randint(1,15)) # 1~15 个反射率
    surf = mean(r, axis=0) # 该点表面光谱 (L,)
    meas = (illum * surf) @ S.T # 投影到 15 通道 → 输入
    gt_wp = normalize(illum @ S.T) # 该点真值白点 → 目标
# 总 set 数 = 32768 * 16 ≈ 524288；每 epoch 重新采样反射率

数据库来源提示：光源 SPD 可用公开库（如 RIT 的 illumination 数据库、或 EU 项目 University of Eastern Finland spectral library）；反射率可用 RIT Munsell /师 等。论文用了"公开 + 内部"混合。

4.2.2 MIMI 验证集采集（硬件协议）

  • 并排固定 RGB 相机 与 MSI 快照模组（论文用 Tab S9 + S1-M），三脚架 + 遥控触发防抖。
  • 每个场景拍两张：无白卡（跑算法）、有白卡 + Macbeth 色卡（量真值）。
  • 白卡贴在平面上，量其 MSI 值作为该处真值白点；Macbeth 用于算 ΔE2000。
  • 人工标注白卡与色卡位置（得到 774 对稀疏白点真值）。

4.2.3 相机 SSF

训练 RGB 版需 Nikon D810（或目标相机）的 SSF；训练 MSI 版需 S1-M 的 15 通道敏感度（论文 Figure 1）。SSF 可来自厂商或 RIT 数据库。MSI→任意 RGB 的转换用最小二乘拟合 S1-M 敏感度到目标 SSF（论文 Figure 6 显示拟合很准）。

4.3 网络实现（PyTorch 草图）

# Set Transformer 四块（简化）
encoder = MLP(d_in, hidden) # 逐点，无交互
aggregate = ISAB(hidden, num_inducing) # Set Transformer 聚合
disperse = ISAB(hidden, num_inducing) # Set Transformer 分发
decoder = MLP(hidden, d_out) # 逐点输出白点

x = encoder(X) # (128, hidden)
x = aggregate(x) # 聚合全局光源信息
x = disperse(x) # 回灌到每点
wp = normalize(decoder(x)) # (128, 15) 每点白点，L2 归一化

控制 2M 参数：hidden ≈ 64–128，inducing points m ≈ 16–32，堆叠 1–2 层 ISAB。相比 LSMI-U 的 56M，这是数量级差距，正是为移动端设计。

4.4 Loss 函数（⚠️ 论文未明确给出，以下为推断）

⚠️ 诚实标注

论文正文没有写出 loss 公式，只说"validation loss"。基于其评估指标是白点角度误差，且输出白点已归一化，最合理的推断是 angular loss：

L = arccos( (p · g) / (‖p‖ · ‖g‖) )
// p: 预测白点(已 L2 归一化) g: 真值白点(已归一化)
// 也可取 1 - cos_sim 作为可微替代，训练更稳定

若同时优化颜色准确度，可能加 ΔE 相关项，但论文未提，故不臆断。

4.5 训练协议

  • Epochs：100。
  • 优化器/学习率：初始 1e-4（Adam 类），若验证损失连续 5 epoch 不显著下降则 减半。
  • 数据增强：① 光源 SPD 的小随机形变（模拟光源微变化）；② 给模拟测量加真实散粒噪声 shot noise（泊松噪声，模拟低光信噪比）。
  • 每 epoch 重采样：打乱 SPD 顺序、重新抽反射率 → 等效无限数据。

4.6 评估协议

  • 指标：白点角度误差的 mean/median；白平衡后 ΔE2000（用 Macbeth）。
  • 跨相机比较：把 MSI 白点转到目标 RGB 相机域后再算角度（Table 4 的 MSI→MSI→RGB）。
  • 注意 Table 2（密集白点图）与 Table 3（稀疏白点）误差口径不同，不能直接横比（论文明确提醒）。
5 论文引用的关键论文深入讲解
你要求"论文引入了其他论文也要分析"。下面把 CIC33 依赖/对照的 5 篇核心引用逐一展开（其余 [1–4][7–9] 在末节简述）。
[12] Set Transformer: A Framework for Permutation-Invariant Neural Networks
Lee J, Lee Y, Kim J, Kosiorek A, Choi S, Teh YW · ICML 2019 · arXiv:1810.00825
网络 Backbone

为什么重要：CIC33 的 aggregation/dispersion 两块直接用它。它解决"输入是一堆没有顺序的东西（set），输出也要与顺序无关"的任务。

核心组件

  • MAB(A,B) = LayerNorm(A + rFF(LayerNorm(A + MHA(A,B))))：标准多头注意力，A 是 query，B 是 key/value。
  • SAB(X) = MAB(X, X)：自注意力（X 同时是 Q/K/V）。
  • ISAB(X) = MAB(X, MAB(I, X))：诱导点 I（可学习的小集合，如 16–32 个）先"压缩"X 的信息，再让 X 向压缩后的 I 做注意力。复杂度从 O(n²) 降到 O(n·m)，且对 X 的行顺序置换不变。这正是 CIC33 想要的：128 个测量打乱顺序，结果只是对应打乱。
  • PMA(X) = MAB(I, SAB(X))：用若干 seed 向量做"池化"，把集合压成固定个数的全局表示。

CIC33 的用法对应：encoder（逐点 MLP）→ aggregation ≈ ISAB（把集合聚成全局光源表示）→ dispersion ≈ ISAB / SAB（把全局表示分发回每点）→ decoder（逐点 MLP）。轻量 + 置换不变 + 线性复杂度，三者兼得，故能在手机跑。

[5] Large Scale Multi-Illuminant (LSMI) Dataset
Kim D, et al. · ICCV 2021, pp.2410–2419 · GitHub
Benchmark + 批判对象

它是什么：第一个大规模、带逐像素密集白点真值的多光源数据集（7486 张 / 2762 场景，3 台相机）。获取真值的方法是差分成像法（differential imaging）：控制场景里"除了一个光源外全灭"，分别拍图，用差分反解每个光源的白点图。

LSMI-U 网络：论文配套的 local WB 网络，56M 参数，multi-scale 结构。它是 CIC33 的 SOTA 对照。

CIC33 如何用它：① 在 LSMI 测试集上 benchmark（Table 2，打平）；② 在 MIMI 上 benchmark（Table 3，SPTY 泛化更优）；③ 批判其成像模型（第 2.4 节）：LSMI 的密集白点真值建立在 RGB 精确的近似上，而该近似对 RGB 不准——所以 LSMI 真值本身有系统误差，训练出的 LSMI-U 泛化差。

[10] Single Pixel Spectral Color Constancy
Koskinen S, Acar E, Kämäräinen JK · IJCV 132:287–299, 2024（华为芬兰 + 坦佩雷）· DOI
多光谱 WB 前身（华为）

它与 CIC33 的关系：这是把多光谱用于白平衡的奠基之作（在我们《华为路线图》文档里详述）。核心发现：

  • 只需 10–14 个光谱通道 即可形成光源的"光谱指纹"；场景平均光谱形态 ≈ 光源光谱。
  • 方法：用 RGB-to-Spectral 把 RGB 上采样成光谱，再估光源；在 235 张真实多光谱图（Mate 20 Pro + 色度计）上验证，比 RGB 方法提升 40–70%。

CIC33 在引言里把它列为"多光谱作为白平衡额外输入"的代表（[9][10][11]），但强调：前人没做"局部"白点估计——这正是 CIC33 的差异化贡献（局部 + 轻量网络 + 真实硬件验证）。

[6] Attentive Illumination Decomposition (AID) for Multi-Illuminant WB
Kim D, et al. · CVPR 2024, pp.25512–25521 · 项目页
Local WB SOTA

它是什么：当前 multi-illuminant / local WB 的强基线。核心思路是用 slot attention 把场景"分解"成若干个（如 2–4 个）代表光源，再学一个"质心匹配损失（centroid-matching loss）"让每个像素归属到合适的光源。输出一张光照图。

与 CIC33 的对照：CIC33 引言把 [5][6] 并列为"local white point 近期兴趣"的代表。区别在于：AID 仍基于 RGB、网络较重；CIC33 主张用多光谱 + 超轻量 Set Transformer 来降歧义、降算力。两者是"同一目标的不同技术路线"（RGB 分解 vs 多光谱感知）。

[11] Beyond RGB: A Real World Dataset for Multispectral Imaging in Mobile Devices
Glatt O, et al. · WACV 2024, pp.4344–4354 · 链接
多光谱移动数据

它是什么：三星的 16 通道 MSI 真实移动数据集（1680 场景，含 36 维真实光谱真值，Samsung/OPPO 手机配对）。和 CIC33 同属"移动多光谱"阵营，但 Beyond RGB 偏"光谱重建/增强"通用任务，CIC33 偏"局部白平衡"特定任务。两者共同说明：移动多光谱硬件（双摄/面阵）正在成为产业共识。

6 总结、局限与未来方向

6.1 论文贡献一句话总结

用一个 2M 参数的 Set Transformer，在 大规模模拟多光谱数据 上训练，实现了：① 在 LSMI 上打平 56M 的 SOTA；② 在真实多样场景上泛化更好；③ 多光谱比 RGB 局部白点更准（角度↓29%、ΔE↓25%）。并提出可公开、可转任意 RGB 相机的 MIMI 验证集。

6.2 局限（阅读时要有保留）

  • 模拟数据简化：无空间/光谱相关性、无复杂光-物质交互，虽"够用"但非真实分布。
  • 稀疏真值：MIMI 只有白卡处的稀疏白点，不是密集白点图——评价口径与 LSMI 不同，且无法精细评局部。
  • 仅 15 通道：相较华为/三星的 16 通道，通道数与几何（S1-M 是快照式滤光片）有特定限制。
  • 未做端到端 CCM：只估白点，没联合颜色校正（对比华为 WACV 2025 的 AWB+CCM 联合优化）。
  • Loss 未公开：复现需自行推断（见 4.4）。

6.3 未来方向（论文自述 + 本系列延伸）

  • 扩展 MIMI + 密集真值协议：论文自己说要"devise a protocol for dense local white point in-the-wild"。
  • 显式建模局部歧义：用多光谱信息建模"白点不确定性"，再结合场景内容（语义） 选实际白点——这会和 AID 的"光源分解 + 语义消歧"思路汇合。
  • 多光谱 + 光源分解（最有前景的融合）：把 AID 式 slot attention 分解，接到多光谱输入上——同时攻击"歧义"与"可解释"，且天然契合华为红枫栈（见《Local AWB 待解难题》第 6 节）。

0什么是 Local AWB：从"全局"到"局部"的范式跃迁

自动白平衡（AWB）的传统假设是整幅图像只受单一、均匀的全局光源支配。Local AWB（局部自动白平衡）打破这一假设：同一场景中不同空间位置可能受不同光源/不同混合比例照射，因此必须为每个像素（或每个局部区域）估计并施加独立的白点增益。

◈ 核心定义

Local AWB = 在图像空间上变化的白点估计。它等价于"多光源（multi-illuminant）场景下的逐像素颜色恒常性"。输出的不是单一的 3 维白点向量，而是一张逐像素光照图（illumination map）——图像上每个位置都对应一个独立的白点（光照色度）。

Global AWB

单光源假设输出 1 个白点

对整幅图估一个对角增益矩阵 D=diag(d_R,d_G,d_B)，全图共享。在 mixed-illumination 场景下，要么夸大某光源色差，要么只部分去偏——"此消彼长"。

Local AWB

多光源/空间变化输出光照图

每个像素/局部区域独立白点 D(x,y)。能正确处理"窗边暖光 + 室内冷光"这类真实场景，但引入局部歧义（regional ambiguity）——区域颜色信息少时，比全局更不确定。

为什么这件事"难且重要"？因为 真实世界绝大多数场景都是多光源的：室内的人造光 + 窗外的日光；户外的直射阳光 + 阴影；舞台的多色灯光。商用相机沿用单光源假设，正是大量"偏色照片"的根源。Local AWB 是多光谱 AWB 与 计算颜色恒常性 两条主线在"空间维"上的交汇点——这也是本系列前文（华为路线、概念辨析）所铺垫的技术终点。

1研究思路与物理基础（原理细节）

任何 Local AWB 方法都建立在同一个成像模型之上。理解这个模型，才能理解所有方法的"能"与"不能"。

1.1局部对角模型与光照图

在 Lambertian 反射假设下，位置 x=(x,y) 处、第 k∈{R,G,B} 通道的观测值为：

// 单光源局部成像模型 f_k(x) = m(x) · e_k · r_k(x) m(x) : 几何明度（shading，与光源颜色无关） e_k : 该位置光源在第 k 通道的响应 ← 要去除的 r_k(x): 物体表面反射率（含 ColorChecker/光谱） ← 要保留的

白平衡的目标就是把 e_k "约掉"。对全局情形，施加对角矩阵 D=diag(d_R,d_G,d_B)，其中 d_k = e_k^c / e_k（e^c 为标准光源）。对局部情形，对角矩阵随位置变化——PWCC（Pixel-wise Color Constancy, Multi-Illuminant, 2024）给出的双光源逐像素增益矩阵为：

// 局部（双光源）von Kries 增益矩阵 I^ab(x,y) = [ e1(x,y) 0 0 0 e2(x,y) 0 0 0 e3(x,y) ] I^ab(x,y) : 逐像素白平衡增益矩阵，即本文的"光照图" e1,e2,e3 : 每个像素、每个通道独立的光照响应
◈ 关键洞察

光照图本质上是一张逐像素的对角增益场。Local AWB 的全部工作，就是从一张 RGB 图（或 MS 图）反演出这张场。反演是 ill-posed 的：从 f_k 无法唯一解出 e_k 与 r_k(x)（二者耦合），必须靠先验或数据驱动。

1.2混合光源模型：每个像素是"多个光源的线性混合"

真实多光源场景下，一个像素往往同时被多个光源照射。最常见的简化是双主光源线性混合模型（Yue & Wei, Optica 2024；也用于 CIC33 的物理后处理）：

// 双主光源混合（逐像素） e(x) ≈ w1(x)·e_a + w2(x)·e_b w1(x)+w2(x) = 1, w1,w2 ≥ 0 e_a, e_b : 两个主光源（场景级，全局恒定） w1,w2(x) : 每个像素的混合权重图（空间变化） ← 真正的未知量

这意味着 Local AWB 可以拆成两个子问题：① 识别场景级存在哪几个光源（它们的色度 e_a, e_b）；② 估计逐像素的混合权重 w1(x), w2(x)。AID（CVPR 2024）正是沿这条思路设计的（见 §3.3）。

更一般地，主-从光源模型（master–subordinate illumination model）（Xue et al., IEEE TCI 2025）把混合光照推广为一种"相关色温再照明"的扩展，用于建模复杂多色温场景。它把物理图像形成显式写成：

// 物理图像形成（混合光照，Xue 2025） f = ∫ ρ(λ) · SPD_mix(λ) · S(λ) dλ SPD_mix(λ) = 主光源 SPD + 从光源 SPD 的加权和 ρ(λ) : 窄带反射率； S(λ) : 相机光谱灵敏度

该模型的价值在于：显式建模"窄带反射率 ↔ 光源 SPD"的相关性，从而能从 RGB（甚至多光谱）恢复场景光的完整 SPD，再把 RGB 与估计光谱融合，精确预测光照色度——这是把"物理先验"重新引入学习型 Local AWB 的代表。

1.3为什么 RGB 做 Local AWB 尤其受限

把全局问题"局部化"后，难度不是线性增长而是阶跃式上升，原因有三：

  • 局部颜色信息更少 → 歧义更大。全局算法可用全图统计（灰世界/白块），但一个局部 patch 的颜色可能极单调（如一面纯色墙），光源估计退化成"猜"。CIC33 明确指出："local estimation ... confronted with even more ambiguity due to its regional nature."
  • 同色异谱（metamerism）在局部被放大。3 通道 RGB 无法区分"真实白点"与"恰好在该区域投影相同"的其他光源。像素级越细，这个歧义越致命。
  • 位深（bit-depth）敏感。ISP 偏好低位深以省算力，但低位深会损失细节、放大噪声，使逐像素算法精度暴跌（详见 §5.3）。

这三条恰好对应多光谱的切入点：多光谱用 N 维（如 15 维）替代 3 维，从根本上压低歧义——这也是 §4 与 CIC33 的核心论点。

2现有解决思路总览：五条主线

把 2014–2026 年的工作按"思路"而非"时间"归类，可归纳出五条主线。它们并非互斥，前沿方法往往是杂交体。

主线 A
局部统计 / 分区法——把图切块，每块当单光源独立估计后融合（patch-based）。代表：Gijsenij 分割+聚类、Beigpour CRF、Bianco patch-CNN、CIC 2008 HDR 局部加权法。优点是可复用全局 AWB；难点是 patch 边界 + 小区域不准 + 融合。
主线 B
逐像素学习（Pixel-wise DNN）——用 U-Net/Transformer 直接回归光照图（image-to-image）。代表：LSMI-U、AngularGAN、TransCC、SMM、PWCC、Yue&Wei(Optica)。理论最优，但算力大、低位深敏感、需密集真值。
主线 C
光源分解（Illumination Decomposition）——先分解成"几个场景级光源 + 逐像素权重"，再融合。代表：AID（slot attention）、Multi-scale 三分支融合。可解释、能给光源数/色度、支持光照编辑。
主线 D
高效 / 解析加速——把全局高效算法（FFCC）扩展为局部平滑光照图。代表：IFFCC（CVPR 2025）。参数量与速度碾压 DNN，精度持平。
主线 E
多光谱 / 融合 / 生成式——用多光谱降歧义（CIC33/MIMI、华为 IJCV2024/WACV2025）、用 sRGB 多预设融合（Mixed/Style WB、Revisiting Fusion ICCV2025）、用扩散模型跨相机（GCC）。最活跃的前沿。

3现有解决方案：方法级解读

3.1局部统计 / 分区法（主线 A）

① 早期分区融合（CIC 2008, HDR 场景）。针对"多光源 HDR"提出局部 AWB：对每个像素，从其邻域的颜色信息估计光照，权重由空间距离、亮度差、色度三者加权。这是最早显式"逐像素、局部加权"的思路，比深度学习早了近十年，但依赖手工权重，鲁棒性差。

② Gijsenij 分割+聚类（2011 survey）。把图分成小 patch，每块做单光源估计，再用聚类得到光照分布；Beigpour 用 CRF 优化 patch 级初值；Bianco 用 CNN 做 patch-wise 估计。它们的问题是patch 边界不连续、忽略空间依赖、无法保证光照平滑过渡。

③ 灰像素 / 灰度法（Gray Pixel, Pixel Greyness）。Gray Pixel 算法找图中的"灰像素"作锚点，再滤波估计每像素光照；Thai et al.(2016) 用 YCbCr 的"灰度"作权重做加权平均。依赖"能否准确找到灰像素"，在单调色场景失效。

3.2逐像素学习（主线 B）

LSMI-U（U-Net）

ICCV 2021数据集+基线

Kim et al. 提出的 U-Net，输入 256×256 RGB，输出逐像素光照图。是后续几乎所有方法的基线。缺点：56M 参数、对 LSMI 过拟合、低位深敏感。

LSMI dataset, ICCV 2021

AngularGAN

GAN里程碑

首个用 GAN 处理 non-uniform / multi-illuminant 的白平衡，引入 angular loss。问题是常引入空间伪影，图像不自然。

multi-illuminant GAN WB

PWCC（TV + 双边滤波）

2024平滑

Pixel-wise Color Constancy via Smoothness：U-Net + Total Variation loss 强制光照图平滑，再用双边滤波保边去噪。针对"光照本应随空间平滑变化"这一先验。还用 label smoothing 缓解 LSMI 真值噪声。

arXiv:2402.02922

TransCC / SMM

Transformer自监督

TransCC 用 transformer 多任务框架；SMM 用自监督 + transformer 编码器，刷新 LSMI 基准。说明全局依赖建模对空间变化光照很重要。

LSMI benchmark SOTA

◈ 重点：低位深鲁棒性（Yue & Wei, Optica 2024）

这是把"Local AWB 落地到 ISP"最关键的一篇工程论文。核心发现：同一逐像素算法在不同位深上精度差异可达 30%（低位深更差），原因是低位深损失细节 + 噪声增大——这一点过去从未被指出。ISP 偏偏偏好低位深（省算力），形成矛盾。

解法 = L1 loss（比 L2 更抗噪/异常值）+ 物理约束后处理。后处理分四步：① 把光照图转到 uv 空间，用 K-means 找两个主光源；② 离散化两主光源连线、以容差圆识别噪声像素；③ 把噪声像素的光照用象限内局部平均替换（基于"光传播近线性"）；④ 据两主光源夹角判断单/多光源。最终比 SOTA DNN 高约 40% 精度。

3.3光源分解（主线 C）—— AID 详解

AID：Attentive Illumination Decomposition Model（CVPR 2024, 延世大学）是当前 Local AWB 的 SOTA 代表，也是与多光谱最易结合的方向。它不像 B 类方法直接回归光照图，而是先把混合光照"分解"成若干场景级光源。

// AID 成像模型（G 通道归一化为 1，只需 R/B 色度） f(x) = Σ_k α_k(x) · ℓ_k · ρ(x) α_k(x) : 第 k 个光源在像素 x 的权重图（softmax 和为 1） ℓ_k : 第 k 个光源的色度（chromaticity，场景级） → 混合光照图 ℓ_mixed(x) = Σ_k α_k(x)·ℓ_k 即输出

实现上，AID 用 Slot Attention（K=7 个 slot）让每个 slot 负责表示一个光源：U-Net 编码器提取与图同分辨率的特征 → 迭代校准（T=3 次）slot → 每个 slot 输出一组 (α_k, ℓ_k) → 融合成 ℓ_mixed。关键在于质心匹配损失（centroid-matching loss）：

// 质心匹配损失：让正确 slot 激活、其余关闭 L_centroid = Σ_i ( |ℓ_i − ℓ̂_{σ_m(i)}| + |α_i − α̂_{σ_m(i)}| ) σ_m : 预测 slot 与 GT 光源的匹配 无此损失 → 所有 slot 都激活、互相干扰；有此损失 → 正确 slot 激活、其余关闭

结果（LSMI 数据集，MAE 角度误差，越低越好）：

Multi-scale Illuminant Estimation and Fusion（Luo et al., 2025）

尺度分解三分支U-Net

动机：光照分布在小尺度更均匀、大尺度更多样，但此前方法忽略尺度依赖。提出把光照图表示为多粒度分量的线性组合：三个 U-Net 分支分别从多尺度图估光照图，再用注意力融合模块生成逐像素权重做线性组合。在 LSMI 上达到 SOTA。

3.4高效加速（主线 D）—— IFFCC 详解

IFFCC：Integral Fast Fourier Color Constancy（CVPR 2025, 中科大 & DJI）解决的是"像素级 DNN 太重、无法实时视频"的痛点。它把为单光源设计的 FFCC（Fast Fourier Color Constancy）扩展为多光源局部版本。

  • 积分 UV 直方图（Integral UV Histogram）：把"任意局部区域的 UV 直方图"用积分图数据结构预计算，提取时变成 O(1) 查表——这是核心加速点（传统要每个 patch 重复算直方图）。
  • 傅里叶卷积并行化：对每个区域做 FFCC 估计后，用傅里叶卷积平滑融合成空间连续的光照图。
  • 结果：参数量仅 1.2 万（比 DNN 少 400×+），256×256 图在 CPU 上 0.03 秒（快 20–100×），精度与像素级神经网络持平或更好。

思路分类上 IFFCC 属于"patch-based + 平滑融合"，但用解析结构替掉了神经网络，是工程落地的关键路径。

3.5融合式方法（主线 E-融合）

思路：不预测光照图，而是把输入图用若干预定义 WB 预设处理成多个 sRGB 版本，再用网络融合。代表：Mixed WB、Style WB。

Arora 硕士论文 (2024)

融合5个预设

用注意力架构融合 5 个 WB 设置，比前作提升达 25%；并渲染了 16,000 张 sRGB 训练集。

Revisiting Image Fusion (ICCV 2025)

Transformer融合+100%

Serrano-Lozano et al. 指出线性融合非最优，提出高效 transformer 捕捉跨 WB 预设的空间依赖，自建 16k sRGB（5 预设）数据集，比已有技术提升最高 100%。

4多光谱 + Local AWB（最相关的前沿）

这是用户研究的核心，也是华为路线与本系列 CIC33 论文的交汇。多光谱通过升维直接攻击 Local AWB 的根本困难——局部歧义。

4.1CIC33：用多光谱提升 Local AWB（Spectricity, CIC 2025）

这正是用户本地的论文。它首次把多光谱用于局部白点检测（此前多光谱仅用于全局白点）。

数据：MIMI 数据集

Spectricity 用自家的 S1-M VIS 快照式多光谱模组（15 通道，SVGA 分辨率，见图 1 光谱响应）采集。核心贡献是 MIMI（Multi-Illuminant Multi-spectral Imaging）数据集：

  • 774 个场景，每个场景用 RGB（Samsung Tab S9） 与 MSI（S1-M） 双模同步采集，含/不含白卡成对图像。
  • 真值用白卡（white charts）测得稀疏局部白点（而非密集光照图）；附 Macbeth 色卡算 ΔE2000。
  • 关键特性：MSI 可高精度转换成任意 RGB 相机的光谱响应（图 6），因此该数据集能生成"任意 RGB 相机的白点真值"——通用性极强。

训练：模拟多光谱测量（避免过拟合）

由于真实 MS 数据稀缺，网络在大规模模拟测量上训练：假设完美 Lambertian，用 32768 条光源 SPD（荧光/白炽/卤素/LED/日光及组合）+ 6814 条反射率光谱投影到相机光谱灵敏度；单/混合光源各半；共生成 524288 组测量集，每 epoch 重排。这一设计让网络不依赖真实数据的场景分布。

网络：Set Transformer（仅 2M 参数）

为适配移动端，用轻量 transformer：编码器（逐测量处理）→ 聚合块（提取光照信息）→ 散列块（把光照信息关联回每个输入）→ 解码器（输出白点）。聚合/散列用 Set Transformer 架构。该方法不限多光谱——同样可训在 RGB 上（RGB→RGB 变体）。

◈ CIC33 的三个核心结论
  1. 多光谱 > RGB：MSI→MSI 相比 RGB→RGB，白点角度误差 ↓约 29%（2.8→2.0），ΔE2000 ↓约 25%（4.8→3.6）。"用多光谱测量去光源，比 RGB 准得多"。
  2. 模拟训练泛化性远好于中等规模真实集：LSMI-U 在 MIMI 跨域测试暴跌（4.4°），证明 LSMI 过拟合；纯模拟训练的 SPTY 跨域仍稳（3.1°）。
  3. 对 LSMI 成像模型的批判：LSMI 用 m = SΛ_r λ ≈ SΛ_r S⁺S λ = D_r l，该近似仅在"SPD 完全落在 S 的列空间 + D_r 为对角"时成立——对 RGB 相机几乎不成立，对多光谱相机则准确得多。这正是多光谱降歧义的物理根源。

4.2华为：单像素光谱恒常性 + 多光谱颜色复制

前文（华为路线图文档）详述的两篇，在本语境下是 Local AWB 的"上游能力"：

  • IJCV 2024（Single Pixel Spectral Color Constancy）：从单像素 MS 数据恢复光源 SPD（10–14 通道即可）。这给 Local AWB 提供了"每像素都能估准光源"的物理基础——把全局高维估计下沉到像素级。
  • WACV 2025（Multi-Spectral Image Color Reproduction）：正则化色彩管线 + 光照自适应 CCM（illuminance-adaptive matrix）。其"按场景光照动态调 CCM"本质上就是局部/条件化颜色校正，是多光谱 Local AWB 在 CCM 环节的体现。

4.3物理 + 学习：主-从光源模型（Xue et al., IEEE TCI 2025）

北航工作把物理渲染 + 神经网络结合做多光源颜色恒常性：显式建模窄带反射率与光源 SPD 的相关性，从 RGB + 光谱恢复完整 SPD，再融 RGB 预测色度。它把"物理可解释"重新拉回学习型框架，是本系列 §6 未来方向的重要参照。

4.4生成式 / 跨相机：GCC（CVPR 2025, MediaTek+NYCU）

虽非直接 Local AWB，但解决了 Local AWB 的孪生难题——跨相机泛化。GCC 用预训练扩散模型inpainting 生成一张符合当前场景光照的色卡，从色卡灰块提取光照。单步确定性推理 + Laplacian 分解保结构 + mask 数据增强。结果：跨相机 worst-25% 误差 5.15°/4.32°，无需传感器专属训练。思路可直接迁移到"逐像素生成局部色卡/光照图"。

5待解难题（深度剖析）

Local AWB 远未解决。以下 8 个难题按"从物理到系统"排序，每个都附当前瓶颈与已有尝试。

① 真值歧义："正确的局部白点"是什么？

混合模型只在"双主光源"下成立，真实可能是 3+ 光源或连续分布；而且物理白点 ≠ 人眼偏好白点（感知 vs 物理之争）。目前没有"in-the-wild 密集真值"的获取协议——CIC33 只能取稀疏白卡真值，并把"建立密集真值协议"列为未来工作。

瓶颈：真值获取本身 open problem

② 互反射与高光（非 Lambertian）

成像模型假设 Lambertian，但镜面高光、次表面散射、互反射（colored illumination from inter-reflection）都违反它。AID 的"彩色光照"仍简化；真正的彩色光照需 intrinsic decomposition 分离。Careaga(2024)、Sha(2025) 的彩色本征分解是方向，但与 AWB 尚未打通。

瓶颈：分解不准确 → 白点错

③ 位深与信噪比

Yue&Wei 证明低位深使逐像素精度跌 30%。ISP 要低位深省算力，形成死结。L1+物理后处理缓解但治标。当 Local AWB 下沉到手机主 ISP，低位深 + 小区域双重噪声是硬约束。

瓶颈：精度↔算力的 trade-off

④ 跨相机 / 跨域泛化

不同传感器 SSF 不同 → 同一场景 RAW 不同 → domain gap。LSMI 过拟合（CIC33 表3）；DMCC 双映射、GCC 扩散跨相机是解法，但 Local AWB 的跨域仍弱。多光谱因"光谱指纹"对 SSF 不敏感，是天然优势（CIC33 表4）。

瓶颈：传感器专属数据需求

⑤ 数据集与真值瓶颈

LSMI：需控制"除一个外所有光源"，真实场景（如窗外日光+阴影）做不到（见图3）；成像模型近似；训练/测试划分差→过拟合。MIMI：仅 774 场景、稀疏真值、无密集图。两者都无法支撑 in-the-wild 密集 Local AWB 训练。

瓶颈：数据规模 + 真值质量

⑥ 光源数未知 & 可解释性

场景有几个光源是未知的。AID 用质心匹配损失让 slot 自适应激活/关闭，是优雅解法，但 slot 数 K 仍须预设。能否端到端推断光源数？以及对"为何这样分解"的可解释性，仍是开放问题。

瓶颈：变长结构化输出

⑦ 算力与移动端部署

像素级 DNN（LSMI-U 56M）无法进手机实时视频。IFFCC(1.2万参数)、SPTY(2M)、set transformer 是轻量方向，但精度与速度的平衡仍在拉锯。视频还需时序一致性（当前研究少）。

瓶颈：实时 + 低功耗

⑧ 多光谱融合的独特难题

多光谱相机成本/尺寸/功耗高；通道数取舍（Spectricity 15 通道快照）；MS→RGB 转换不完美（尤其荧光灯）；如何把 MS 局部白点与主 RGB 成像对齐融合（手机多光谱多为辅助测光，不参与主成像）。华为红枫路线正在攻这一问题。

瓶颈：sensor + 系统级集成

⚠ 最深的一层矛盾

Local AWB 的"局部"既是解法也是根源。局部化让我们能处理多光源，但也让每个估计单元可用的颜色信息骤减，把"ill-posed 反问题"推向更病态。所有技术（多光谱、物理先验、拆分光源、上下文语义）本质上都在给局部单元补回信息——这是理解全部待解难题的统一视角。

6未来研究方向探索

6.1物理先验 + 学习型（Hybrid）

Xue 2025 的主-从模型指明方向：把物理图像形成（窄带反射率↔SPD 相关性）作为归纳偏置嵌入网络，而非纯数据驱动。优点是样本效率高、可解释、跨域稳。下一步是把这种物理约束下沉到逐像素，并与 AID 式分解结合。

6.2多光谱 + 局部分解（最有前景的融合）

把 AID 的 slot attention 接到多光谱输入：slot 负责表示场景级多光谱光源，权重图在光谱空间估混合——既保留 AID 的可解释/光照编辑能力，又用多光谱压低局部歧义（呼应 CIC33 表4 的 29%/25% 提升）。这是当前最接近"完美 Local AWB"的组合，也是华为 IJCV2024(WB) + WACV2025(CCM) + 红枫传感器可拼出的完整栈。

图 6-1 · "多光谱 + AID 式分解"融合架构示意（最有前景方向）

6.3生成式 / 扩散模型

GCC 证明扩散先验可用于光照估计与跨相机泛化。未来：① 逐像素生成光照图（扩散去噪光照场）；② 用扩散模型做物理约束的数据增强（模拟任意混合光源的密集真值）；③ 把"局部白点歧义"显式建模为扩散的不确定性（CIC33 未来工作之一）。

6.4Transformer / 时空建模

SAIGFormer（2025，低光）用"动态积分图 + 空间自适应光照估计器 + 光照引导注意力"处理非均匀光照，证明 transformer 对空间变化照明有效。多尺度融合（Luo 2025）也指向"尺度感知"。视频 Local AWB 还需时序一致性建模（当前几乎空白）。

6.5上下文 / 语义辅助消歧

局部歧义的本质是"信息不足"，补信息的另一来源是语义与元数据：① 前后景分割（人脸/天空）→ 分区融合；② Time-Aware AWB（Samsung, 2025）：用拍摄时间戳 + 地理位置作为光照先验（~5K 参数，正午/日出/日落 CCT 范围不同）；③ 场景理解网络选择"实际局部白点"。CIC33 也把"用互补信息（场景内容）选实际局部白点"列为未来工作。

6.6自监督 / 仿真到真实

SMM 已证自监督在 LSMI 有效。CIC33 用大规模物理模拟训练取得最佳泛化（表3），指明"sim-to-real + domain adaptation"是突破数据瓶颈的主路。下一步：把模拟的光照混合、互反射、高光都建模进去，缩小 sim-real gap。

6.7评测体系革新

当前指标（角度误差、ΔE2000）不足以刻画"局部自然度"。需要：① 感知指标 + 用户研究（LSMI 已有 user-study，应扩展）；② 密集真值协议（in-the-wild）；③ 区分"物理正确"与"观感偏好"的双重评测。

◇ 一张图看未来收敛点

Local AWB 的最终形态很可能是：多光谱传感（降歧义） + 光源分解网络（可解释） + 物理渲染约束（跨域/样本效率） + 语义/元数据上下文（消局部歧义） + 轻量实时架构（移动端） 的融合体。它不是某一个新模型，而是上述主线的合流。

7总结

Local AWB 是颜色恒常性从"全局单光源"走向"空间变化多光源"的必然演进，也是多光谱 AWB 与 计算摄影 在像素维的交汇。本文按"思路而非时间"梳理了五条主线：

  • A 局部统计（patch/加权）——最早但受限于 patch 边界与局部歧义；
  • B 逐像素 DNN（LSMI-U/TransCC/PWCC/Yue&Wei）——理论最优但重、敏感于位深；
  • C 光源分解（AID slot attention、多尺度融合）——可解释、能给光源数/色度；
  • D 高效加速（IFFCC 积分 UV 直方图）——参数量/速度碾压 DNN；
  • E 多光谱/融合/生成（CIC33/MIMI、华为双论文、Fusion、GCC）——当前最活跃前沿。

待解难题集中在八个层面：真值歧义、非 Lambertian（互反射/高光）、位深噪声、跨域泛化、数据瓶颈、光源数未知、移动端算力、多光谱系统集成。最值得投入的方向是 多光谱 + AID 式光源分解 的融合——它同时攻击"歧义"与"可解释"两个核心，且与华为"红枫传感器 + IJCV2024(WB) + WACV2025(CCM)"的布局天然契合。

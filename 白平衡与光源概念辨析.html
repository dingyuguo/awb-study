白平衡与光源：从RGB到多光谱的概念辨析

解开"恢复灰 / 恢复标准光源 / 预测光谱 / 预测白点"四种说法的统一框架
白平衡 WB 颜色恒常性 多光谱 AWB 概念辨析 2026-07-11
序 你真正困惑的是什么
讨论里反复出现四种关于"白平衡到底是什么"的说法，它们层次不同、坐标系不同，所以显得互相矛盾。
说法 A

白平衡是"把灰色恢复成灰色"。—— 直觉上很对，但灰色恢复成灰色并不等于完成了白平衡。

说法 B

白平衡是"恢复到标准光源"。—— 学术定义，但"恢复标准光源"听起来和"恢复灰"像是两件事。

说法 C

多光谱里要做的是"预测光源光谱（SPD）"。—— 这比白平衡听起来"高级"很多，是同一种任务吗？

说法 D

多光谱里要做的是"预测白平衡点"。—— 那白平衡点和光源光谱到底是不是一回事？能互相转换吗？

本篇要证明的论点

这四种说法不是矛盾，而是同一件事在不同层次、不同坐标系下的表述。统一框架是：

白平衡的本质 = 估计光源 → 用对角矩阵把图像从"未知光源"变换到"标准光源"下。

白平衡点永远是"标准光源 ÷ 估计光源"（逐元素比值）；这个关系对 RGB 和 MS 都成立。真正改变的是sensor 空间的维度——从 3 维升到 16 维，信息损失从严重变为可忽略。这才是多光谱 AWB 优于 RGB AWB 的根本原因。

1 统一框架：白平衡 = 变换到标准光源
先把最上层的概念钉死，下面所有讨论才有锚点。

1.1 WB 与 CCM：两个不同层次的任务

ISP pipeline 里 WB 在前、CCM 在后，它们的目的不同：

步骤数学形式要解决的问题
WB 白平衡对角缩放
R'=g_R·R 等		光偏了——消除光源色温造成的整体偏色，让中性物体恢复中性
CCM 颜色校正3×3 矩阵
R'=Σm·RGB		相机跟人眼不一样——补偿 sensor 光谱灵敏度与人眼 CIE CMF 的差异（通道互相混合）

顺序不能反：CCM 是在"标准光源"空间中做 sensor→人眼的映射，它假设输入已经白平衡。如果先做 CCM，光源色偏会污染矩阵求解。所以铁律是 WB 先归一化光源 → CCM 再做色彩映射。

1.2 "恢复灰" vs "恢复标准光源"：不矛盾，只是层次不同

这两个说法都对，但描述不同层面：

"恢复到标准光源""恢复到灰色"
白平衡的正式目标定义 (a) 一种光源估计手段（灰世界假设）；(b) 白平衡正确后的验证标准
学术定义：变换到 canonical illuminant 下的外观 中性面（灰卡）在标准光源下本就是中性的，所以"灰恢复成灰"是结果

关键结论

"标准光源"是定义，"灰色恢复成灰色"是手段 + 结果。两者不矛盾——在标准光源下，灰卡本来就该是灰的。灰色是白平衡的"症状"，不是"病因"。

权威依据：Gijsenij et al.（IEEE TIP 2011）明确写道——"the aim of most of the color constancy methods is to transform all colors of the input image... to colors as they appear under this canonical light source"。而 gray-world 被定义为一种光源估计的假设（"the average reflectance in a scene under a neutral light source is achromatic"），不是白平衡的定义本身。

2 机制拆解：灰世界 → 对角矩阵 → 白点
理解了"目标"之后，要搞清"怎么做"。这里最容易混淆"RGB 等值"到底指什么。

2.1 图像形成模型（一切的起点）

在 Lambertian 反射假设下，第 k 通道（k∈{R,G,B}）的成像可分解：

f_k(x) = m(x) · e_k · r_k(x) // m=几何明度(与颜色无关) e_k=光源在k通道响应(要去掉) r_k(x)=物体反射率(要保留)

白平衡的全部目标，就是约掉 e_k，恢复 r_k(x)——让图像呈现物体"在被人眼适应的光源下"该有的颜色。

2.2 灰世界假设：只是第一步——估计光源

灰世界假设认为场景平均反射率是无色的，于是图像各通道均值就近似等于光源颜色：

ê_k = mean( f_k(x) ) // 估计出的光源颜色（第k通道）

这只是诊断（知道光是什么颜色），还没治疗。Gray world 只是众多光源估计算法之一，同类还有 White Patch、Gray Edge、Gamut Mapping、深度学习等。

2.3 对角矩阵 = 标准光源 ÷ 估计光源

估计出 ê_k 后，构造对角矩阵（von Kries 模型）：

d_k = e_k^canonical / ê_k // 白平衡点（gains）的第k个元素
若 canonical = 白光(等值) → d_k = 1 / ê_k （归一化后）

应用到图像后，估计光源被约掉：

f_k^corrected(x) = d_k · f_k(x)
= (e_k^canonical / ê_k) · m(x) · ê_k · r_k(x)
= m(x) · e_k^canonical · r_k(x) // ê_k 消去 → 只剩"标准光源下的物体"

核心关系（务必记住）

白平衡点 = 标准光源 ÷ 估计光源（逐元素倒数）。 光源估计回答"当前光是什么颜色"，对角矩阵回答"要乘什么系数把它拉到标准白光"。前者是诊断，后者是处方，两者是倒数关系。

2.4 回头看"让 RGB 三通道等值"

"RGB 等值"可以指三件完全不同的事，只有分清对象才不会乱：

哪种"RGB 等值"含义在白平衡中的角色
(a) 光源的 RGB 等值e_R=e_G=e_B，光源本身是中性白光这是标准光源的定义——参考目标
(b) 图像均值的 RGB 等值mean(R)=mean(G)=mean(B)这是灰世界假设——估计光源的手段
(c) 每个像素的 RGB 等值所有像素 R=G=B摧毁所有颜色！绝对不是 WB

所以"让 RGB 等值 ≠ 恢复标准光源"。白平衡完成后，只有中性面的 RGB 等值；红色苹果做完 WB 仍是红的（它的反射率 r_k 本来就非等值）。

3 坐标系统的转折：RGB 的 3 维 vs 多光谱的 N 维
这是整篇的关键。第 2 节的"白点 = 标准光源 ÷ 估计光源"对 RGB 和 MS 都成立。真正改变的是 sensor 空间的维度。

3.1 一个被忽略的前提：白点维度 = sensor 维度

传统 RGB 相机的白平衡点是 3 维 (d_R, d_G, d_B)；16 通道多光谱相机的白平衡点是 16 维 (d_1, ..., d_16)。这不是巧合——白平衡点永远是光源在"当前 sensor 空间"里的表示，所以它的维度 = sensor 通道数。

3.2 维度决定了信息损失

RGB（3 通道）多光谱（16 通道）
白平衡点维度316
在 sensor 空间内 白点=光源？✅ 是的（逐元素倒数）✅ 是的（逐元素倒数）
sensor 空间能否代表真实 SPD？严重信息损失近似无损

3.3 为什么 RGB 会"丢信息"：同色异谱（Metamerism）

不同形状的真实光源 SPD，只要它们在相机 RGB 响应上的投影相同，就会得到完全相同的 3 维白平衡点。但它们对物体颜色的还原可能差异很大。

关键引用

Gao & Mantiuk（Cambridge, 2025）："Because different illuminants can have the same projection in the 3-dimensional SSFs space, those traditional methods cannot differentiate between such illuminants (metamers in the camera response space)."

所以：多光谱 AWB 的优势不是"白点↔光源的关系变了"，而是 sensor 空间从 3 维升到 16 维，使得在投影到 3 维 RGB 之前就能完成光源估计，避免了不可逆的信息坍缩。

4 灰卡论证：在 MS 空间，"白点 ≡ 光源"
这是讨论最后推导到的核心洞察，也是把前面所有概念串起来的那一步。

4.1 灰卡成像与光源平行

灰卡反射率是一个常数 s(x,λ)=c。代入图像形成模型：

f_k^gray = m·c·∫ e(λ)·ρ_k(λ) dλ = m·c·e_k

即 f^gray = (mc) · e——灰卡成像向量与光源向量平行，角度完全一致，只差一个标量倍数 mc。你最初提出的"角度一致"判断完全正确。

4.2 推论：MS 白点 = MS 光源的逐元素倒数

MS 白点 d_k = e_k^canonical / ê_k，而 ê_k ∝ f_k^gray ∝ e_k，故：

d_k ∝ 1 / e_k // MS 白平衡点 = MS 光源的逐元素倒数，信息完全等价

结论

在 MS sensor 空间内，预测白平衡点 = 预测光源（两者信息等价，仅取倒数）。你推导到这一步的直觉是对的。

4.3 那"预测光源光谱"和"预测白点"到底是不是一回事？

问题答案
在 MS sensor 空间内，白点=光源？是（逐元素倒数，等价）
"白点"是连续 SPD e(λ) 吗？不是，它是投影到 MS sensor 的 16 维向量
16维 → 连续 SPD 可逆吗？近似可逆（well-posed，16个窄带通道覆盖可见光谱）
3维(RGB) → 连续 SPD 可逆吗？不可逆（ill-posed，无穷多 SPD 投影到同一3维点）

所以精确地说：

  • RGB 里"预测白点"丢掉了光谱细节，无法区分同色异谱光源。
  • MS 里"预测白点"几乎等于"预测光源"——只是坐标系在 sensor 空间而非物理 SPD 空间。若想进一步得到连续 SPD，可由 16 维做光谱重建（近似可逆）。

华为 IJCV 2024"单像素光谱颜色恒常"本质上就是：在高维 MS 空间恢复光源（等价于预测 MS 白点），再经光谱重建得到连续 SPD。如果只有 3 通道 RGB，这一步在数学上就做不到。

5 落到研究：概念如何连回多光谱 AWB 论文
把上述框架应用到你研究的核心论文，看它们各自"卡"在框架的哪一环。

5.1 华为 IJCV 2024（单像素光谱颜色恒常）

在高维 MS 空间从单个像素恢复光源 SPD。对应框架的"预测 MS 光源"——利用 16 维信息避免 RGB 的 3 维坍缩，从而能从单像素（而非整图统计）估光。

5.2 华为 WACV 2025（AWB + CCM 联合优化）

把 WB gains 与 CCM 端到端联合学习，直接输出 CIE1931 XYZ。对应框架的"在高维空间完成 WB+CCM"——既然 WB 与 CCM 都依赖光源估计，且 MS 空间光源是高维的，联合优化比"先估 3 维白点、再查表做 CCM"更准，尤其在多光源/混合光源下。

5.3 CIC33（Spectricity, Local AWB）

用 MS 图像做每个像素的光源估计（local AWB）。对应框架"白点是 per-pixel 的高维光源"——16 维让每个像素的光源估计比 RGB 的 3 维更可信，前文所述 MSI 相比 RGB 角度误差降低 29%、ΔE2000 降低 25% 正源于此。

一句话总结多光谱 AWB 的本质

多光谱 AWB 不是魔法，而是"在投影到 3 维 RGB 之前，先在高维 MS 空间完成光源估计"。白点与光源的逐元素倒数关系没有改变，改变的是 sensor 空间维度——从 3 升到 16，信息损失从"严重、不可逆"变为"可忽略、近似可逆"。

6 公式速查
把全文关键公式汇总，便于随手查阅。
① 图像形成（Lambertian）: f_k(x) = m(x) · e_k · r_k(x)
② 灰世界光源估计: ê_k = mean( f_k(x) )
③ 对角矩阵(白点): d_k = e_k^canonical / ê_k （canonical=白光时 d_k=1/ê_k）
④ 校正后图像: f_k^c = d_k·f_k = m·e_canonical·r （ê_k 被约掉）
⑤ 多光谱白点↔光源: d_k ∝ 1/e_k （在 MS sensor 空间内等价）
⑥ 灰卡成像: f_k^gray = m·c·e_k （与光源向量平行，同角度）
